# Frontend

This package contains the React SPA for Docgate. It is the user-facing control panel around the protected content site: sign-in, email verification UI, purchase flow, dashboard, and admin operations all live here.

The frontend does not serve the protected docs directly. Instead, it handles account and purchase workflows, then sends users back to Nginx-controlled routes where access is enforced.

## Responsibilities

- render SuperTokens auth pages
- show the user dashboard and payment history
- allow prepaid-code redemption
- start and complete the Stripe checkout flow
- expose admin-only tools
- redirect users out of the SPA to protected docs routes

## Tech Stack

- React 19
- React Router 7
- Vite
- TypeScript
- SuperTokens Auth React
- Stripe React SDK
- Tailwind CSS + DaisyUI

## App Structure

```text
src/
  app/
    dashboard/        user profile and payment history
    manage/           admin tools
    purchase/         prepaid code + Stripe purchase entry
    stripe/           checkout and return flow
    jump-out-spa/     redirect back to Nginx-controlled site routes
  supertokens/        auth setup and route guards
  utils/api.ts        backend API client wrappers
  routes.ts           SPA routes + API route mapping
  config.tsx          env-backed runtime config
```

## Important Pages

- `dashboard`: shows user email, tier, lifetime, and pay log.
- `purchase`: supports both prepaid-code redemption and Stripe checkout.
- `manage`: admin-only page for operational actions.
- `jump-out-spa`: hands routing back to the main site so Nginx can decide whether docs access is allowed.

## Admin Features

The manage page currently exposes:

- prepaid code generation
- password reset link generation
- manual email verification

Route access is guarded with SuperTokens session claims and requires the `admin` role.

## How It Connects to the Rest of the Stack

```text
React SPA
  -> calls FastAPI /api/... endpoints
  -> uses SuperTokens session cookies
  -> redirects users to Nginx-managed docs routes
  -> Nginx re-checks access before serving protected content
```

## Environment

The frontend reads runtime values from Vite env files generated by `confgen`, including:

- website base paths
- API domain and base path
- auth route base path
- Stripe publishable key
- product/site naming values

The Vite `base` option is derived from `VITE_WEBSITE_REACT_BASE_PATH`, so the SPA can be mounted under a subpath such as `/app/`.
